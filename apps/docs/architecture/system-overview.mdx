## Layers

**Solana program** (`programs/blinkremit`): escrow PDAs, `create_escrow`, `claim_escrow`, `refund_escrow`. Enforces mint, token program, relayer pubkey, and ed25519-based authorization payload verified inside `claim_escrow`.

**API / orchestration** (`apps/web/app/api`, `apps/web/lib`): builds serialized transactions for employers and relayer-paid claim txs; verifies wallet-signed employer JWT mint flow (`JWT_SECRET`) and contractor session JWT (`SESSION_SECRET`, after email OTP verification stored in Redis).

**Persistence**: Postgres via Prisma — `Employer`, `Blink`, `Credential`, `OfframpRequest`. Blink rows carry `status`, optional `escrowPDA`, `escrowTxSig`, `claimTxSig`, contractor wallet + credential (off-chain claim secret for relayer authorization).

**Async**: Redis for BullMQ queue `offramp` (`lib/queue.ts`, `workers/offramp.ts`), Redis for contractor OTP and queues (`lib/redis.ts`), cron routes gated by `CRON_SECRET`.

**Webhooks**: Helius and Meso routes verify HMAC on raw bodies (`lib/webhookHmac.ts`), then patch DB rows conservatively.

## Request shape

The product is **event-driven**: UI calls APIs that mutate Blink rows and occasionally enqueue jobs or confirm chain state. The authoritative DB enum aligns with `lib/blinkStateMachine.ts` and Prisma `BlinkStatus`; illegal transitions return HTTP 409 with `INVALID_STATE_TRANSITION`.

## Environment seams

Critical env vars include `PROGRAM_ID`, `USDC_MINT_ADDRESS` (must match program constants for non-custom deployments), `RELAYER_KEYPAIR` / `RELAYER_PUBKEY`, RPC URLs, `DATABASE_URL`, `REDIS_URL`, JWT/session secrets, and webhook secrets. See `apps/web/.env.example`.
